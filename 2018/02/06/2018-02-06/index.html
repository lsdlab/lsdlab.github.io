提升开发效率和体验的 SASS 工具/服务推荐

2018-02-06

推荐一些我在用的和看起来不错的 SASS 工具，有开发用的也有增长用的一些服务，好服务当然也是要付费的，而且还不便宜，但是好的工具和服务用起来就是舒服，效率也很高。

线上监控

  • http://nodequery.com 一个简单的服务器监控报警服务，需要在 root 下安装 agent，每九十秒上报一次信息，功能很简单，免费账户能添加十个服务器，我也不在用了，现在的 VPS 服务商都有自己的监控提醒了，DigitalOcean 也能添加监控规则，能发邮件或者 Slack 通知。
  • https://amplify.nginx.com/overview/ Nginx 监控，能看 Nginx 的性能数据，请求次数等等。
  • https://my-netdata.io/ 开源监控工具，黑色背景大屏看起来非常帅，安装升级都很简单，支持物联网设备，底层用 C 写成，所以不会非常消耗太多 CPU 和内存。
  • https://hyperping.io ping 监控，免费账户三个域名，这种 ping 监控服务太多了，这个算是便宜的。
  • https://lsdlab.statusticker.com/ 监控各种 SASS 服务的服务，有人想知道自己在用的各种服务是不是正常的吧。

异常捕获及 APM

  • http://sentry.io 异常捕获，支持各种框架的前后端应用，免费账户一万条消息，对于个人小项目也够用了，不想买服务的话也能自己部署。
  • https://opbeat.com 应用性能分析服务，APM 服务国内国外都有很多，国外有名的就是 datadog/new relic，这个也不错。
  • https://www.datadoghq.com/
  • https://newrelic.com/ datadog 和 new relic 算是国外比较有名的产品了，商用付费应该不错，公司用可以考虑，个人就不用考虑了，太贵。

增长工具

如果你有开发新的产品，想看用户的使用情况、一些指标，除了 Google Analysis 和国内的什么 CNZZ 百度，但是国内真的别用，都不是什么好东西，尽量用 GA，增长黑客里面倒是推荐了好多个这样的工具，当然，都是要付费的。

  • https://mixpanel.com/
  • https://www.gosquared.com/
  • https://www.kissmetrics.com/

搜索

如果有比较重的搜索需求，又不想用 ES 作为搜索后端的化用 https://www.algolia.com/ 好像也不错的样子。
