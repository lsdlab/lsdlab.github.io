又部署了一遍 Django

Posted by Chen Jian on Fri 29 July 2016 Updated on Fri 29 July 2016

Django + Nginx + uwsgi 部署

Django -- 1.9.8 Nginx -- 1.4.6 uwsgi -- 2.0.13.1

ubuntu 上要依赖三个东西 python-dev, libxml2-dev, pcre。

libxml2-dev 是用来解析 uwsgi 的 .xml 脚本的，pcre 是 Perl 的正则表达式解析库，Django 解析 URL 的，python-dev 我也不知道是干什么的

Nginx 开个 80 端口，uwsgi_pass 传给 8001 这个端口，uwsgi 在通过这里把 Django 跑起来。

1. Nginx 配置 /etc/nginx/sites-enabled/yunka_pool_web.conf

server {
       # the port your site will be served on
       listen      80;
       # the domain name it will serve for
       server_name 163.44.151.44; # substitute your machine's IP address or FQDN
       charset     utf-8;

       # max upload size
       client_max_body_size 75M;   # adjust to taste

       # logs
       access_log /home/chen/yunka_pool_web/logs/access.log;
       error_log /home/chen/yunka_pool_web/logs/error.log;

       # Django media
       location /media {
               alias /home/chen/yunka_pool_web/media; # your Django project's media files - amend as required
       }

       location /static {
           alias /home/chen/yunka_pool_web/static; # your Django project's static files - amend as required
       }

       # Finally, send all non-media requests to the Django server.
       location / {
               uwsgi_pass  163.44.151.44:8001;
               include    /home/chen/yunka_pool_web/uwsgi_params; # the uwsgi_params file you installed
   }
}

2. Django 项目下需要从 Nginx 源码中复制出一份 uwsgi_params

uwsgi_param  QUERY_STRING       $query_string;
uwsgi_param  REQUEST_METHOD     $request_method;
uwsgi_param  CONTENT_TYPE       $content_type;
uwsgi_param  CONTENT_LENGTH     $content_length;

uwsgi_param  REQUEST_URI        $request_uri;
uwsgi_param  PATH_INFO          $document_uri;
uwsgi_param  DOCUMENT_ROOT      $document_root;
uwsgi_param  SERVER_PROTOCOL    $server_protocol;
uwsgi_param  REQUEST_SCHEME     $scheme;
uwsgi_param  HTTPS              $https if_not_empty;

uwsgi_param  REMOTE_ADDR        $remote_addr;
uwsgi_param  REMOTE_PORT        $remote_port;
uwsgi_param  SERVER_PORT        $server_port;
uwsgi_param  SERVER_NAME        $server_name;

3. Django 项目下需要一个 yunka_pool_web_wsgi.py

# -*- coding: utf-8 -*-

import os
import sys

reload(sys)
sys.setdefaultencoding('utf8')

from django.core.wsgi import get_wsgi_application    # Django 1.7 前是 from django.core.handlers.wsgi import WSGIHandler

os.environ.setdefault("DJANGO_SETTINGS_MODULE", "yunka_pool_web.settings")    # “你的项目.settings”
application = get_wsgi_application()    # 1.7以前是 application = WSGIHandler()

4. uwsgi 启动脚本 yunka_pool_web_socket.xml 放在 Django 项目下

<uwsgi>
       <socket>163.44.151.44:8001</socket>
       <chdir>/home/chen/yunka_pool_web</chdir>
       <module>yunka_pool_web_wsgi</module>
       <processes>4</processes>
       <daemonize>/home/chen/yunka_pool_web/uwsgi.log</daemonize>
</uwsgi>

最后重启 Nginx，启动 uwsgi 的脚本 load Django App

sudo nginx -s reload
uwsgi -x  yunka_pool_web_socket.xml
